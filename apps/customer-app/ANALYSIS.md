## 6. Repositories & Data Sources

### Repository Pattern
Each domain has an **abstract repository** (in `domain/repositories/`) and a **concrete implementation** (in `data/repositories/`) that delegates to one of two datasource implementations.

### Dual Datasource Architecture

```
                        ┌─ appwrite/*_datasource.dart  (PRIMARY, useAppwrite=true)
Repository Impl ──────┤
                        └─ remote/*_datasource.dart    (LEGACY, Dio/REST fallback)
```

The `useAppwrite` flag in each repository impl determines which datasource is used. Currently hardcoded to `true`.

## 13. Issues, Bugs & Incomplete Items

### Critical Issues

1. **`DropdownButtonFormField` uses `initialValue` instead of `value`** — This property doesn't exist on `DropdownButtonFormField`. Appears in `saved_addresses_screen.dart` (both add and edit dialogs) and `location_selection_screen.dart`. Will cause a compile error or runtime issue.

2. **`Radio` widget in `payment_methods_screen.dart` uses `RadioGroup<String>` pattern** — There's no standard Flutter `RadioGroup` widget. The `Radio<String>` widgets inside use a `groupValue` but the `RadioGroup` wrapper is non-standard and likely won't compile without a custom implementation.

3. **Chat screen bypasses BLoC architecture** — Makes direct Appwrite SDK calls from the widget. Mixes UI and data concerns, makes testing difficult, inconsistent with the rest of the app.

4. **No error handling for Appwrite Realtime disconnections** — All screens subscribe to Realtime channels but don't handle reconnection on network loss. The chat screen has a 5s polling fallback, but other screens don't.

### Missing Backend Integration

5. **Wallet screen** — 100% dummy data. No backend calls. Balance, transactions, add money, withdraw, promo codes all non-functional.

6. **Payment Methods screen** — All "coming soon". Card storage, JazzCash, Easypaisa integration not implemented.

7. **Notification Settings** — Saved only to SharedPreferences locally. Backend is not notified of user's notification preferences, so push notifications can't be filtered server-side.

8. **Payment gateway** — No payment processing integration exists anywhere (Stripe, JazzCash, Easypaisa, etc.).

### Architectural Issues

9. **`core/config/` directory is empty** — Seems intended for environment configuration but unused.

10. **Home screen uses `findAncestorStateOfType<MainScreenState>`** for cross-tab navigation — Tight coupling between HomeScreen and MainScreen. Should use a callback or shared state.

11. **No domain entities or use cases** — `domain/` only has repository interfaces. The Clean Architecture domain layer is incomplete (no entity classes, no use case classes). BLoCs call repositories directly.

12. **Hardcoded strings persist in many screens** despite l10n file having 161 strings. Profile screen, chat screen, SOS screen, booking tracking screen all contain hardcoded English text.

13. **`email_input_screen.dart`** — Misleading filename. The screen handles phone number input, not email. Should be `phone_input_screen.dart`.

14. **Inconsistent city lists** — Location selection screen uses 8 cities, saved addresses screen uses 12 cities. Should use a single source of truth.

### Missing Features

15. **Voice input** — `speech_to_text` is commented out in pubspec. Service selection screen shows "coming soon" placeholder.

16. **Image attachment in chat** — Shows "Attachment feature coming soon" snackbar.

17. **Payment flow after booking completion** — Booking tracking shows "Pay Now" but no payment screen or flow exists.

18. **Worker profile view** — Referenced in worker cards but no dedicated worker profile screen exists.

19. **Booking dispute flow** — `disputed` status exists in the model but no UI to raise or manage disputes.

20. **Admin panel integration** — No admin-specific features in the customer app (expected, but some admin notification handling is incomplete).

### Performance Concerns

21. **No image caching strategy** — `cached_network_image` is in pubspec but profile images are loaded via raw Appwrite URLs without consistent cache configuration.

22. **Booking list loads all bookings per tab** — No proper pagination on initial load (only pull-to-refresh). Large booking histories will be slow.

23. **Worker location polling in chat** — 5-second Timer.periodic runs indefinitely while chat is open, even when not needed.

### Missing Files / Infrastructure

24. **No `.env` file or environment configuration** — Appwrite config is hardcoded in `appwrite_config.dart`.

25. **No CI/CD configuration** — No GitHub Actions, Fastlane, or other deployment pipeline.

26. **No code generation setup** — Despite having `retrofit` in dependencies, no `.g.dart` generated files are present. The Dio/REST datasources appear to use manual HTTP calls.

27. **Assets are placeholders only** — All referenced images, animations, and SVG icons in `AppAssets` don't actually exist. Only README files in asset directories.
